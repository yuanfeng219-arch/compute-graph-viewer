Whitepaper · VLSI EDA

布局算法如何实现芯片“核舟记”

VLSI placement 的任务，是把逻辑综合后的门级网表、宏单元和 FPGA 资源实例，放进有限芯片区域里，并同时压住线长、时序、拥塞、功耗、合法位置和后续布线可行性。它不是简单排版，而是一个带物理约束、离散约束和多目标反馈的大规模优化问题。

内容来源： 基于 Obsidian 笔记《设计VLSI EDA(7)布局算法怎么实现芯片“核舟记”》重组整理，并结合 PTO 既有 H-Anchor / HNSW 白皮书中关于 N-body、coarse-to-fine、anchor graph 与 incremental update 的讨论。
千万级现代布局对象可以达到千万级器件和线网规模，运行效率本身就是核心约束。
HPWL半周线长是布局阶段最常用的快速线长估计，但需要可导近似才能进入解析优化。
Timing关键路径 slack 会改变边权，让时序紧张的实例更强地互相靠近。
LegalFPGA 还要满足离散资源位置、LUT/FF 打包和资源兼容性约束。

Definition · 布局到底在放什么

ASIC 与 FPGA 的共同点是“图到几何”，差异在物理约束

布局输入通常是逻辑综合后的网表：实例是点，net 是连接。输出是芯片平面上的位置。ASIC 更像连续区域上的标准单元排列；FPGA 则必须映射到预先存在的 LUT、FF、BRAM、DSP、时钟和互连资源上，因此离散性和 packing 约束更强。

ASIC：连续排布 + 行对齐

标准单元在行里摆放，目标是密集、短线、低功耗、时序闭合。宏单元和电源网络会显著影响可用空间。

Models · 线长和时序怎么被估计

布局阶段不能等布线完成，必须用可快速更新的代理指标

布局算法每轮都会移动大量实例，如果每移动一次都调用完整 routing，成本不可接受。因此 EDA 工具会使用线长和时序的近似模型，在精度、速度和可导性之间取平衡。

HPWL

半周线长只看一个 net 的最左、最右、最上、最下引脚，计算快，常作为全局布局的主代理。

HPWL = (max x - min x) + (max y - min y)

Star / RSMT

星形线对 timing 更友好；RSMT 更接近真实布线，但需要专门算法，通常比 HPWL 慢。

Star ModelSteiner Tree

Timing Graph

时序分析把电路转成 DAG，通过正向 arrival 和反向 required 计算 slack，进而改变关键边权。

SlackCritical Path

Floorplanning · 先决定大方向

好的初始版图，是让后续优化少翻山越岭

FPGA 资源分布不均，ASIC 也会受宏单元、电源网和 IO 约束影响。若初始布局把强相关模块放到不合适的远端区域，后续力导向或解析优化很容易陷入局部最优，或者用大量迭代才能修正。

Netlist Partitioning

把网表分成若干子图，目标是减少跨分区连接、控制每个分区资源需求、保持分区规模均衡。传统方法包括 KL、FM，工程里也会使用 PaToH、KaHyPar 等 hypergraph partitioner。

Cluster Placement

分区之后决定 cluster 的相对位置。小规模 cluster 可以用 simulated annealing；层次化场景也可以用 slicing tree 或递归切分。

Global Placement · 解析模型和 N-body

全局布局把“拉近连接”与“避免堆叠”写进同一个优化问题

现代全局布局常用解析法：把每个实例位置视作变量，把线长、时序、密度、锚点约束等写成可求导或可迭代近似的目标函数，然后用数值求解器不断更新坐标。直觉上，net 像吸引力，密度和重叠像排斥力，资源地形像外部势场。

Quadratic / Bound2Bound

把不可导的绝对值线长近似成二次项，通过动态更新权重逼近 HPWL，适合线性代数求解器。

Timing-driven Pseudo Net

对 slack 紧张的路径插入或加重伪连边，把关键路径局部推成更短、更快的几何形状。

Anchor / Preconditioning

锚点防止所有节点被吸到一起；预条件让梯度地形更均匀，减少求解器在狭长峡谷中来回摆动。

低保真图 · N-body placement 直觉
Density · 扩散和面积修正

布局质量不只看线短，还要给布线和打包留下空间

如果只优化线长，所有互连强的实例都倾向于挤在一起。器件扩散、兼容性面积修正和拥塞面积修正，就是把“放得下、能打包、走得通线”这些后续约束提前反馈到布局阶段。

Cell Spreading

从 overflow 区域找一个资源足够的窗口，把实例在窗口中重新摊开。它比排斥力更直接，但移动也更硬。

Packing-aware Area

不兼容的 LUT/FF 不能随便塞进同一个 CLB，因此实际资源需求会变大，并随邻居关系动态变化。

Congestion-aware Area

高连接密度区域需要更多 routing 通道，布局器会把这些区域的单元“变大”，为布线预留空间。

Legalization · 从连续坐标回到物理位置

全局布局给方向，合法化负责落地

解析布局中的坐标通常是连续的，但芯片上的可放置位置是离散的，FPGA 尤其明显。合法化需要把实例映射到合法 site，同时尽量不破坏全局布局已经获得的线长和时序优势。

一对一匹配

简化场景可以抽象成二分图匹配：左边是实例，右边是 site，边权是移动距离或 QoR 代价。

一对多与 packing

FPGA 中某些实例占用多个位置，或者多个 LUT/FF 共享一个 CLB，需要动态规划或更复杂的组合优化。

Detailed Placement

最后扫描关键路径和局部毛刺，通过 swap、候选窗口、最短路径模型改善 WNS/TNS 等指标。

Product / Research Insight · 对 PTO 与 H-Anchor 的启发

Placement 的难点在于“全局导航”和“局部修正”必须同时成立

这篇笔记给 PTO / H-Anchor 的核心启发是：布局问题天然像大规模 N-body 系统，但工程上不能真的全量互相作用。可行路径是用层次化结构先建立稀疏全局骨架，再在局部进行高精度修正。

布局环节算法含义对 PTO / H-Anchor 的启发
Netlist partitioning 先切分强相关子图，减少跨区域优化压力。 大图执行、算子调度和 placement 原型都应先抽取 cluster / anchor。
Analytical placement 把线长、时序、密度写成统一可优化目标。 性能、通信、内存局部性可以变成边权和势能项，而不是只做静态检索。
Cell spreading 解决全局优化后局部过密的问题。 调度也需要处理执行资源热点，不只是把相关节点放近。
Legalization / packing 把连续理想解映射到真实硬件资源。 任何 HNSW-like / H-Anchor 方法最后都要回到 Ascend / FPGA / runtime 的真实资源约束。
Detailed placement 扫描关键路径并做局部重定位。 增量 ECO、性能回归定位和关键链路修复，是产品层最容易落地的能力。
结论：VLSI placement 不是“把点画漂亮”，而是在不完整物理信息下，用代理模型持续逼近可布线、可时序闭合、可制造的真实版图。它对 HNSW / H-Anchor 的启发不是“拿 HNSW 做向量索引”，而是借用高层稀疏导航、底层局部细化和增量更新的工程结构，把全局复杂度拆成可控的多层优化流程。
Framework · 核心模型

把 placement 看成“代理模型 + 层次优化 + 物理落地”的框架

这篇笔记最适合沉淀成一个三层框架：上层把真实物理问题压缩成可快速评估的代理指标，中层用层次化和解析优化搜索全局结构，底层再把理想坐标投影回真实硬件约束。

Findings · 必须记住的结论

布局难在多目标互相拉扯，而不是坐标计算本身

从产品和研究角度看，placement 的价值在于把复杂物理约束前置到可优化的图模型中，让系统能在还没布线、还没完全合法化之前，就持续做出接近最终 QoR 的判断。

核心张力线长越短不一定越好；过度靠近会带来 density overflow、routing congestion 和 packing 失败。
优化方向用 timing slack、density、resource demand 和 congestion feedback 动态调权，而不是固定一套边权。
局限性代理模型永远不等于最终 routing，placement 必须接受后端反馈并保留增量修正能力。

短期建议

01
保留代理指标解释每个布局决策都能追溯到 HPWL、slack、density 或 congestion 中的一个原因。
02
把关键路径单独可视化全局指标变好不代表最差路径变好，必须让 WNS/TNS 风险可见。

长期方向

03
引入层次化 anchor先用高层稀疏骨架导航，再在底层做局部 N-body / detailed placement。
04
做增量 ECO 闭环支持小范围网表变化、约束变化或时序反馈后的局部重布局。
Terms · 术语表

阅读这篇白皮书需要记住的术语

这些词构成 placement 的基本语言：它们分别描述输入对象、代理目标、优化算法和物理落地约束。

术语含义在布局中的作用
Netlist逻辑综合后的实例与连接关系。placement 的输入图。
HPWLHalf-Perimeter Wirelength，半周线长。快速估计 net 线长的常用代理指标。
Timing Slack信号到达时间与要求时间之间的裕量。slack 越小，相关路径越需要优先优化。
Floorplanning大块电路或宏单元的初始规划。决定全局搜索起点，影响局部最优风险。
Analytical Placement把布局写成可求解的数学优化问题。现代全局布局的主流路径之一。
Cell Spreading把过密区域中的实例向外扩散。降低 density overflow，为布线和合法化留空间。
Legalization把连续坐标映射到合法物理位置。确保布局满足 site、row、资源和重叠约束。
Packing把多个兼容实例组合进一个 FPGA 资源块。FPGA placement 中影响面积、合法化和性能的关键约束。
Detailed Placement在合法布局基础上做局部微调。修复关键路径、局部线长和最后的 QoR 毛刺。
