Chiplet 架构

多 Compute DIE + IO DIE，D2D 超大带宽；双 DIE 形态支持 UMA（Unified Memory Access）。

AI Core = Cube + Vector 混合通道 CV Mix 1:2；SIMT 与 SIMD 在同一 Core 上共存。

CCU — 集合通信单元

零拷贝（zero-copy）集合通信；确定性计算；Reduce 绕过 CCL Buffer 直接写目的端。

AllGather + MatMul 融合可降低 ~70% 通信开销；MoE Dispatch & Combine 融合 +20%。

Regbase 架构

乱序双发射（OOO dual-issue）；Regbase 操作数直接来自寄存器，大幅降低 Local Buffer 带宽压力。

NDDMA：Non-Data-Dependent DMA，支持预取；128B Sector-cache 优化非对齐访问。

新数据类型速览
类型 格式 最大值 主要用途 注意事项
MXFP4 4-bit，有共享缩放因子 — MatMul 权重量化，最激进压缩 需要 Scale 预存入 UB，否则成为瓶颈
MXFP8 8-bit，有共享缩放因子 ~57344 MoE Activation 量化（A5 量化方案） 动态范围远大于标准 FP8-E4M3（max 448）
HiF8 8-bit 特殊格式 ~32768 训练激活值，高动态范围 非标准 FP8，转换时需特殊处理
FP8-E4M3 1+4+3 448 推理权重，兼容 GPU 动态范围较小，LLM 激活值易溢出
FP8-E5M2 1+5+2 57344 训练梯度 精度低于 E4M3，适合梯度存储
存储层次（从外到内）
GM (HBM) → L3 → L2 (512KB) → L1 → L0A/B (64KB) → L0C (256KB) → UB / Reg

关键新路径：L0C ↔ UB 直连（无需绕道 L1），减少 Cube→Vector 数据传输延迟。Vector SIMT 单元独立拥有 Warp Scheduler / Divergence Stack / Register File。

采集整网 Profile

采集全网络算子耗时、通信、内存，得到宏观瓶颈概览。支持 16 进程并发（A2/A3 仅 1 进程）。
02

msprof + MindStudio Insight
定位慢算子、找通信 / 计算重叠问题；看 Timeline、算子摘要、核间负载均衡、Roofline。
03
单算子精细分析
msprof op + MindStudio Insight
对单个慢算子采集 Pipe 流水图 / PC Sampling / SIMT GPR 分析，找 bubble 和 stall。
整网采集命令
# 基础整网采集
msprof \
  --output=./prof_output \
  --application="python train.py" \
  --task-block=on       # 950新增：AICore首尾block
  --sys-hardware-mem=on # 950新增：片上带宽采集
  --process-num=16      # 950支持16进程

# CCU 通信 profiling
msprof \
  --ccu-profiling=on \  # 950新增：CCU Timeline+BW
  --output=./ccu_prof \
  --application="python train.py"
单算子采集命令
# 单算子精细采集
msprof op \
  --op-name=MatMul \
  --pipe-view=on \      # Pipe流水图 (ID14/ID15)
  --pc-sampling=on \    # 9种stall状态采样
  --simt-gpr=on \       # SIMT寄存器分析
  --simt-ipc=on \       # Core Occupancy+IPC
  --output=./op_prof

# 解析
msprof parse --input=./op_prof

减少 GPR 占用防溢出；指令精简去冗余；循环展开提高指令密度；Vector-Fusion 合并多个向量操作。⚠ 已知痛点：mhc_post（↗）、apply_adam_w_v2（↗）等纯 vector 算子实测确认 vf 融合未触发，性能不理想；PyPTO DFX 目前无直接可视化入口确认融合状态。

合并小粒度访存；UB 缓存复用数据；128B 对齐访问避免 sector 浪费；NDDMA 预取隐藏延迟；L2 Cache 提升重用率

950 SIMT 编程 — 5 大优化技巧
①
合理切分线程数（512~1024 最佳）
线程数不超 2048，优先用 b32 类型（b64 占两个寄存器，消耗翻倍）。切分过少浪费并行度；切分过多超出寄存器限制触发 spill。
②
访存合并（Coalesced Memory Access）
同一 Warp 内所有线程的 GM 访问请求落在同一 Sector 时可合并为单次内存请求，带宽利用率最高。最差情况：每个线程访不同 Sector，n 线程 = n 次独立请求。
推荐：output[threadIdx.x] = input[threadIdx.x] // 连续访问，同 Sector
③
避免 Warp Divergence（分支在 Warp 间，不在 Warp 内）
SIMT 硬件为每个 Warp 分配单个 PC，Warp 内分支跳转导致两路串行执行。
✗ 错：if (threadIdx.x % 2 == 0)（Warp 内分支，半 Warp 空转）
✓ 对：int warp_id = threadIdx.x/32; if (warp_id % 2 == 0)（整 Warp 分支）
④
代码优化：合并独立操作利于编译器并发
将独立的 Load/Compute/Store 重排分组：先批量 LOAD，再批量 PROCESS，再批量 STORE。编译器更容易检测到无数据依赖并发指令。
⑤
#pragma unroll 与参数量控制
#pragma unroll：提示编译器展开循环（减少分支 + 提高指令并行）；但会增加代码体积和寄存器消耗，权衡使用。
SIMT 入参最大 28×32 Bit；超过时用预留空间中转，但延迟增大。建议将 tiling 参数结构体搬入 UB 展开。

Ascend C SIMD/SIMT 范式选错导致性能下降
