我对 claude.txt 的理解

这不是运行数据，也不是 PTO 现有 viewer 的输入文件，而是一份单文件 React 原型。它想表达的是：把一个算子的 DAG 拓扑、执行时间、核分配、读写强度和诊断提示放进同一张图里。

换句话说，这份原型试图把 PTO 现有的“开发 / 编译 / 运行”三段视角，压缩成一个围绕单算子的解释界面。
