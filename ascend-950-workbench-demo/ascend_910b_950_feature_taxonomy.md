# 910B / 950 算子迁移特征全量分类

> 配套文档：[`ascend_910b_950_operator_diff.md`](./ascend_910b_950_operator_diff.md)
>
> 这份文档把 md 里出现的所有"950 关键词"剥到本质，归类成 **9 类硬件/软件差异**，并给出反向映射：算子里命中哪些信号 → 多半属于哪一类（A/B/C）迁移场景。

---

## 一句话本质

> 950 是 **新一代芯片硬件 + 新一代编程模型**：AIV 核从 *以内存为中心* 升级到 *以寄存器为中心*；Vector 核多了一套 *SIMT 执行子系统*；Cube 核的物理数据通路被重接、移除了几条旧路径，并新增了 MX 低比特格式。所谓"950 关键词"都是这些底层变化的**表面信号**。

---

## 类别 1 · 代际声明信号（你自己告诉编译器走哪一代）

> 本质：纯**声明性**，没有硬件含义；只决定"用哪套 ISA / kernel 二进制"。看到这些只能判断"算子主观上想跑哪一代"，不能判断它实际能不能跑。

| 信号 | 出现处 | 含义 | 指向 |
|---|---|---|---|
| `CMAKE_ASC_ARCHITECTURES=dav-2201, dav-3510` | CMake | 同一算子双架构编译 | 可能 A |
| `--npu-arch=dav-3510` 固定 | CMake / 编译参数 | 只编 950 二进制 | 偏 B |
| `--npu-arch=dav-2201` 固定 | CMake | 只编 A2/A3 二进制 | 偏 C（待迁移）|
| `__NPU_ARCH__ == 2201 / 3510` | Device 代码 `#if` | Device 侧代际隔离宏 | 已有分支：检查各分支 |
| `SocVersion::Ascend910B / Ascend950PR_9599` | Host tiling 代码 | Host 侧代际隔离 | 同上 |
| `AddConfig("ascend910b")` 注册 | `op_host/*_def.cpp` | 只注册旧代 SoC | 偏 C |
| `AddConfig("ascend950", regbaseCfg)` 注册 | 同上 | 注册 950 实现且带 RegBase 配置 | 偏 B |
| 同时注册 910b + 950 | 同上 | 双代际实现并存 | 多代际共维护 |

---

## 类别 2 · AIV 核：Membase → RegBase（**编程对象**变了）

> **本质**：2201 的 AIV 把"UB 里的 LocalTensor"当作计算对象，指令隐式从 UB 读写；3510 的 AIV 多了一组**矢量寄存器堆**，可以把数据加载到寄存器，**在寄存器上**做计算 / mask / 同步，再写回。
>
> 类比：从"操作变量"变成"操作寄存器"——更接近裸 ISA。

| 指令对象 / 关键词 | 它是什么 | 2201 上 | 3510 上 | 提示 |
|---|---|---|---|---|
| `RegBase` | 一种**编程模型**总称，不是单个对象 | 不存在 | 是 950 AIV 的新模型 | B |
| `RegTensor` | **矢量寄存器张量** —— 寄存器堆里一段固定长度的向量 | 无 | 计算的第一类对象 | B |
| `MaskReg` | **谓词寄存器** —— 一组 bit，控制矢量指令对哪些 lane 生效 | 无（用 selector 间接做） | 显式独立寄存器，可单独算/搬 | B |
| `AddrReg` | **地址寄存器** —— 存指针/偏移，给寄存器搬运指令用 | 无（地址藏在 LocalTensor 里）| 显式对象 | B |
| `LoadAlign` / `StoreAlign` | UB↔RegTensor **对齐搬运指令** | 无 | 寄存器编程的入/出口 | B |
| `LocalMemBar` | **本地内存屏障** —— 寄存器与 UB 间的内存可见性同步 | 用 `set_flag/wait_flag` 等旧原语 | 与寄存器编程配套的新屏障 | B |
| `asc_add`（寄存器级）| **寄存器算子** | 无（`Add` 作用在 LocalTensor 上） | 作用在 RegTensor 上 | B |

> 一句话：看到 `RegXxx / LoadAlign / LocalMemBar`，就知道这是"裸寄存器"风格 → 它本身就是 950 原生写法。

---

## 类别 3 · Vector 核：纯 SIMD → SIMD+SIMT（**执行模型**变了）

> **本质**：SIMD 是"一指令一段连续向量"，没有"线程"概念；SIMT 是"一指令多个独立线程"，每个 lane 有自己的 thread id，可以分歧、可以做线程级 atomic。950 在 Vector 单元内**新增**了 SIMT 子系统，不是替换 SIMD。

| 关键词 | 它是什么 | 2201 上 | 3510 上 | 提示 |
|---|---|---|---|---|
| `--enable-simt` | **SIMT 编译开关**，编译模式而非普通宏 | 不存在 | 必须显式打开才能用 SIMT | B |
| Warp / ThreadBlock | **线程束 / 线程块**（GPU 同名概念）| 无 | 950 SIMT 的并发单位 | B |
| `asc_atomic_add` | **线程级原子加** —— 多线程对同一地址并发累加 | 无 | SIMT 写直方图等场景 | B |
| Gather / Scatter / 直方图 类样例 | **离散访存**场景（适合 SIMT） | 用 SIMD 模拟 | 用 SIMT 自然表达 | B |

---

## 类别 4 · Cube 数据通路（**芯片内互连**变了）

> **本质**：Cube 周围那些 buffer (GM / L1 / L0A / L0B / L0C / UB / Fixpipe) 之间的**物理连线**重接了。这是 die 上的硬件电路变化，不是 API 变。一些旧通路**没有了**，一些新通路**多了**。

| 关键词 / 通路 | 它是什么 | 2201 | 3510 | 提示 |
|---|---|---|---|---|
| `LoadData` | **数据搬运指令**（GM/L1 → L0A/L0B）| 通用 | 仍存在，但 GM→L0 直通已删 | C |
| `Fixpipe` | **Cube 后处理流水**：L0C→GM/UB，含量化/激活 | 通用 | 仍存在；通路增加了 L0C→UB | C |
| **删** GM → L0A/L0B 直通 | 旧 `LoadData` 的快捷路径 | 有 | 没了，必须 GM→L1→L0 | C |
| **删** L1 → GM 通路 | L1 直接写回 GM | 有 | 没了，要走 L0C/Fixpipe | C |
| **新增** UB → L1 | 减少 UB→GM→L1 的绕路 | 无 | 有 | 优化机会 |
| **新增** L0C → UB | 矩阵后处理少一次绕路 | 无 | 有 | 优化机会 |
| `LoadData` + MicroScaling | `LoadData` 的新参数路径，配合 MX 格式 | 不支持 | 新增 | B |
| ND-DMA / loop mode / DN 分型 | **高维 DMA 指令**，搬多维 tensor 更灵活 | 受限 | 新增 | 优化机会 |

---

## 类别 5 · Cube ISA（**计算指令**移除/新增）

> **本质**：Cube 单元的硬件电路上**移除了几条流水**（int4 乘法路径、稀疏路径、L1 边界绕回），同时**新增**了 MX 低比特路径。这是 Cube 内部 die 面积/电路的重新规划。

| 关键词 | 它是什么 | 2201 | 3510 | 提示 |
|---|---|---|---|---|
| `Mmad` | **矩阵乘累加**主指令 | 通用 | 仍是主指令，但操作的格式/分形变 | C |
| `MmadWithSparse` | **稀疏矩阵乘**（4:2 结构化稀疏） | 支持 | **不支持** | C |
| `LoadDataWithSparse` | 配套稀疏搬运 | 支持 | **不支持** | C |
| `int4b_t` Cube Matmul | **4 比特整数矩阵乘** | 支持 | **Cube 不支持**，要 Cast 到 int8 | C |
| `SetLoadDataBoundary` | **L1 边界绕回设定** —— 让 LoadData 在 L1 边界自动绕回 | 支持 | **删除**，要手工拆 Load | C |
| `CheckLocalMemoryIA` | 调试 API：本地内存非法访问检查 | 支持 | **删除** | C |

---

## 类别 6 · 矩阵分形（**Cube 喂数排布**变了）

> **本质**：Cube 要求 A/B/C 矩阵在 L0 里以特定形状摆放（`ZZ`/`ZN`/`NZ` 是不同的 z 字/n 字小块排列）。3510 把 **L0A 的排布从 ZZ 改成 NZ**，L0B/L0C 不变。

| 矩阵 | 2201 分形 | 3510 分形 | 影响 |
|---|---|---|---|
| A (在 L0A) | `ZZ` | **`NZ`** | 写死 L0A 切分/地址计算的代码必改 |
| B (在 L0B) | `ZN` | `ZN` | 不变 |
| C (在 L0C) | `NZ` | `NZ` | 不变 |

提示：C 类。非 L0A 切分场景可能兼容；L0A 切分场景一定要改。

---

## 类别 7 · 低比特数据格式（**新增** MX 系列）

> **本质**：MX = MicroScaling，一组带共享 scale 的低比特格式（FP4/FP8）。Cube/Matmul 高阶 API 新增了原生支持，配套 `LoadData` 有 MicroScaling 扩展。

| 关键词 | 它是什么 | 2201 | 3510 | 提示 |
|---|---|---|---|---|
| `MXFP4` | 4 比特浮点 + 共享 scale | 无 | 原生 Cube 路径 | B |
| `MXFP8` | 8 比特浮点 + 共享 scale | 无 | 原生 | B |
| `FP8-MXFP4` / `MXFP8-MXFP4` Hybrid | 混合精度 Matmul | 无 | DeepSeek-V4 路径 | B |
| `MicroScaling` (在 `LoadData` 上) | 搬数据时附带 scale 处理 | 无 | 新增 | B |
| `Histograms` 指令 | 用于 LightningIndexer Top-k 等 | 无 | 950 专用 | B |

---

## 类别 8 · UB / SRAM 微架构（**bank 拓扑**变了）

> **本质**：UB 是片上 SRAM，分 bank 是为了并发访问。bank 数量、组数、单 bank 容量都变了 → 旧的"错开地址避免冲突"经验**地址公式不一样了**。能编译不代表访问模式还高效。

| 维度 | 2201 | 3510 |
|---|---|---|
| bank group 数 | 16 | 8 |
| 每组 bank 数 | 3 | 2 |
| 每个 bank 容量 | 4 KB | 16 KB |
| 单 UB 总容量 | 16×3×4 = 192 KB | 8×2×16 = 256 KB |

写死 UB 大小 / bank 错位策略的代码（典型 C 类）必查。

---

## 类别 9 · 浮点 / 同步 / 调试语义

| 关键词 | 类别 | 2201 | 3510 | 提示 |
|---|---|---|---|---|
| Subnormal 浮点支持 | **数值语义** | 默认支持 | **默认不支持**，影响 `Exp/Ln/Reciprocal/Sqrt/Rsqrt/Div` | A/C 都可能踩到精度 |
| `Mutex`（核间） | **同步原语** | 无 | 新增 | 优化机会 |
| `CheckLocalMemoryIA` | **调试 API** | 有 | 删除 | C 路径要换 |

---

## 反向映射 · 9 类信号 → A/B/C 结论

| 算子里只命中… | 多半属于 | 因为 |
|---|---|---|
| 仅类别 1 的 `dav-2201,dav-3510` 双声明 | **A** | 主观就奔着跨架构去 |
| 类别 2（RegBase 全家）+ 1 的 `dav-3510` | **B** | 950 原生编程对象 |
| 类别 3（SIMT）| **B** | 950 原生执行模型 |
| 类别 7（MX/Histograms）| **B** | 950 原生 ISA |
| 类别 4 / 5 / 6 任一 | **C** | 触到了 Cube 硬件假设 |
| 类别 8（UB bank 写死）| **C** | SRAM 微架构假设 |
| 类别 9 的 Subnormal / IA 检查 | **C**（兼 A 精度风险） | 数值/调试假设 |

---

## 三类迁移场景速查

| 类 | 含义 | 典型动作 |
|---|---|---|
| **A** · 简单 SIMD 样例 | 弱硬件假设，只用 GM↔UB + 逐元素 | 改 CMake 重编译 → 验精度（Subnormal）→ 验性能（UB bank） |
| **B** · 950 原生能力 | 用了 RegBase / SIMT / MX | 默认按 950 专用看；要兼容 2201 = 另写 2201 分支 |
| **C** · 写死了 A2/A3 硬件假设 | 用了 Cube 路径 / int4 / 稀疏 / 写死 UB | 必须改实现：重排数据通路 / Cast / 重算 tiling |
